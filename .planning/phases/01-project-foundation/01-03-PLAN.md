<task type="auto" tdd="true">
  <name>Task 1: CrawlerService base infrastructure with Isolate and rate limiting</name>
  <files>lib/services/crawler_service.dart, test/services/crawler_service_test.dart</files>
  <behavior>
  - crawlCategory() executes parsing logic inside Isolate.run() (off main thread)
  - Between each HTTP request, enforces minimum 500ms delay
  - Maximum 10 articles per batch (stops after 10 even if more URLs available)
  - HTTP requests use dio with 10s connect timeout, 30s receive timeout
  - Retries failed HTTP requests up to 2 times before skipping article
  - Sets User-Agent header to mimic browser
  - Returns CrawlResult containing List of Article (successes) and List of String (errors)
  - If an individual article parse fails, it is added to errors list and batch continues
  - If listing page fetch fails, returns empty CrawlResult with error message
  </behavior>
  <action>
  Create lib/services/crawler_service.dart:

  Define abstract class SourceCrawler with methods:
  - List of String parseListingPage(String html) — extract article URLs from listing HTML
  - Article? parseArticlePage(String html, String articleUrl, String category) — extract article data from article HTML

  Define CrawlResult class:
  - final List of Article articles
  - final List of String errors
  - Constructor and convenience getters (hasErrors, successCount)

  Define CrawlerService class:
  - Constructor takes SourceCrawler instance
  - Future of CrawlResult crawlCategory(String listingUrl, String category) method:
    1. Wrap entire logic in Isolate.run() — note: dio must be created INSIDE isolate (not passed in)
    2. Create dio instance inside isolate with BaseOptions(connectTimeout: Duration(seconds: 10), receiveTimeout: Duration(seconds: 30), headers: {'User-Agent': 'Mozilla/5.0 ...'})
    3. Fetch listing page HTML
    4. Call sourceCrawler.parseListingPage(html) to get article URLs
    5. Take first 10 URLs only (batch limit)
    6. For each URL: delay 500ms, fetch article HTML with retry (max 2 attempts), call sourceCrawler.parseArticlePage, collect Article or error
    7. Return CrawlResult

  Note: Since SourceCrawler must be passed to Isolate, it must be a top-level function or static. Use a strategy pattern where parse functions are passed as top-level function references to Isolate.run.

  Create test/services/crawler_service_test.dart:
  - Create MockHttpClientAdapter to intercept dio requests
  - Test: crawlCategory with 3 articles returns 3 Articles in result
  - Test: crawlCategory with 15 URLs only processes first 10
  - Test: rate limiting — measure elapsed time for 3 requests >= 1000ms (2 delays)
  - Test: HTTP failure with retry — first attempt fails, second succeeds
  - Test: article parse failure — batch continues, error recorded
  - Test: listing page fetch failure — returns empty result with error
  </action>
  <verify>
    <automated>flutter test test/services/crawler_service_test.dart</automated>
  </verify>
  <done>CrawlerService runs in Isolate. Rate limiting verified (500ms between requests). Batch limit of 10 enforced. Retry logic works. Partial failures handled gracefully. All tests pass.</done>

| T-01-06 | Denial of Service | Rate limiting bypass | mitigate | Hard-coded 500ms delay and batch limit of 10 articles. Cannot be overridden by external input. |
