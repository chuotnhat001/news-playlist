<task type="auto" tdd="true">
  <name>Task 1: ContentService orchestration — crawl + cache pipeline</name>
  <files>lib/services/content_service.dart</files>
  <behavior>
  - getArticles(category) returns cached articles when cache is fresh (no HTTP triggered)
  - getArticles(category) triggers crawl when cache is stale, stores results, returns fresh articles
  - refreshCategory(category) always triggers crawl regardless of cache state (for pull-to-refresh)
  - Supports both 'dantri' and 'soha' sources via source registry
  - Returns combined articles from all sources for a given category, sorted by publishedAt DESC
  - Handles crawl errors gracefully: returns stale cache if crawl fails, empty list if no cache exists
  - Runs clearExpired() on cache during initialization to keep database lean
  </behavior>
  <action>
  Create lib/services/content_service.dart:

  Class ContentService with constructor accepting CacheService and a Map of source name to CrawlerService instances (pre-configured with their respective SourceCrawler).

  Define category-to-URL mapping as a static const Map. Categories: 'cong-nghe', 'kinh-doanh', 'chung-khoan'. Each category maps to listing URLs per source (dantri URL, soha URL).

  Methods:
  - Future of List of Article getArticles(String category):
    1. Check cacheService.isStale(category)
    2. If NOT stale: return cacheService.getArticlesByCategory(category)
    3. If stale: call _crawlAndCache(category), return fresh results
    4. On crawl error: fallback to cacheService.getArticlesByCategory (may be stale but better than empty)

  - Future of List of Article refreshCategory(String category):
    1. Always call _crawlAndCache(category) regardless of staleness
    2. Return fresh results

  - Future of List of Article _crawlAndCache(String category):
    1. For each source in crawlerServices: call crawlerService.crawlCategory(listingUrl, category)
    2. Collect all CrawlResults
    3. Combine all articles from all sources
    4. Call cacheService.insertArticles(combined)
    5. Return combined list sorted by publishedAt DESC
    6. Log errors from CrawlResult.errors (print for now, proper logging later)

  - Future of void init():
    1. Call cacheService.init()
    2. Call cacheService.clearExpired()

  Import CacheService, CrawlerService, DantriCrawler, SohaCrawler, Article.
  </action>
  <verify>
    <automated>flutter analyze lib/services/content_service.dart</automated>
  </verify>
  <done>ContentService compiles. Orchestrates crawl+cache with TTL-aware refresh. Supports multiple sources. Falls back to stale cache on crawl failure.</done>
</task>
